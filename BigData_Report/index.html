TỔNG QUAN VỀ DỰ ÁN

Chào mừng bạn đến với hệ thống hướng dẫn triển khai và vận hành TeleDrive kết hợp Hadoop!

Dự án này nhằm mục đích tận dụng API của Telegram để làm không gian lưu trữ dữ liệu lớn (Big Data Storage), sau đó sử dụng cụm máy chủ Hadoop Cluster để phân tích dữ liệu đó.

Mục tiêu: Xây dựng giải pháp Big Data chi phí thấp (0 đồng cho lưu trữ).
1. HƯỚNG DẪN CÀI ĐẶT TELEDRIVE

TeleDrive đóng vai trò là lớp lưu trữ (Storage Layer). Dưới đây là cấu hình Docker Compose để triển khai.

Cấu hình docker-compose.yml
version: '3.9'
services:
  teldrive:
    image: ghcr.io/tgdrive/teldrive:latest
    command: >
      run
      --tg-app-id=YOUR_APP_ID
      --tg-app-hash=YOUR_APP_HASH
      --jwt-secret="secret"
      --tg-uploads-encryption-key="khoa_bao_mat_32_ky_tu"
    ports:
      - "8080:8080"
    # ... (Các cấu hình khác)

2. HỆ THỐNG HADOOP CLUSTER

Chúng tôi sử dụng phiên bản Hadoop 3.2.1 chạy trên nền tảng Docker để xử lý dữ liệu.

  • Namenode: Quản lý metadata.
  • Datanode: Lưu trữ block dữ liệu.
  • ResourceManager (YARN): Quản lý tài nguyên tính toán.

3. KẾT QUẢ THỰC NGHIỆM (MAPREDUCE)

Chạy thử nghiệm bài toán tính số Pi (Monte Carlo) trên YARN Cluster.

Lệnh thực thi: yarn jar hadoop-mapreduce-examples.jar pi 2 100
